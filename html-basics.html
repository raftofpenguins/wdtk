Basic Tags and Structure of HTML

The following sections indroduce basic topics relevant to the creation of virtually any web page:

  • Getting Started
  • Head
  • Body

Getting Started with HTML

  • Beginning to work with HTML is as simple as opening a text editor, typing anything in it, and saving it as a ".html" file.
  • However, if you want to create a website with more than text displayed, you're going to start needing "tags".
  • The basic format of a tag looks something like this:

    <tag>
    • This tag contains a word enclosed in brackets -- nothing fancy.
  • Many tags you will see often come in pairs, which look like this:

    <tag></tag>
    • The pair terminates with another similar tag, but the second tag has a "/" in front of the name.
    • The pair of tag allows you to designate where something begins and where it ends.
  • This first tag and following pairs of tags are fundamental to virtually any web page:

    <!Doctype HTML>
    <html>
    
        <head>
        
        </head>
    
        <body>
        
        </body>

    </html>

The Head

  • The "Head" contains several elements about the website that aren't always seen by viewers.
  • Information about the HTML document, including the title, links to scripts, and other "metadata" appear in the Head.
  • Giving your web page a title of "Penguin Rafting Adventures" would look like this:

    <head>
        <title>Penguin Rafting Adventures</title>
    </head>
    • This makes "Penguin Rafting Adventures" appear on your web browser's tab as the name of your web page when you open it.
    • The title also affects various aspects of how search engines find your web page, along with other metadata.
  • It is also useful to link to documents containing scripts in the Head, rather than including lines of code within the HTML document.
  • The types of scripts commonly linked to include JavaScript, CSS, etc.
  • Linking to a CSS document generally looks something like this:

    <head>
        <link rel="stylesheet" href="stylesheet.css">
    <meta name="viewport" content="width=device-width" />
    </head>
    • Provided you have a document called "stylesheet.css" in the same folder as your web page, you will now be able to make changes to your CSS document that affect your website!
    • For more information on CSS, go here.
  • Linking to a JavaScript document generally looks something like this:

    <head>
        <script type="text/javascript" src="script.js"></script>
    </head>
    • Provided you have a document called "script.js" in the same folder as your web page, you will now be able to make changes to your JavaScript document that affect your website!
    • For more information on JavaScript, go here.

The Body

  • Now we can start creating elements that viewers can actually see!
  • The "Body" of an HTML document contains the structure for a website, such as the content you're reading now.
  • Text is sufficient for a simple website, but HTML allows us to create much more dynamic websites with which users can interact.
  • To create a hyperlink, which takes the viewer to another page or website altogether once clicked, use this tag:

    <body>
        <a href="http://www.google.com">This is a hyperlink to Google!</a>
    </body>
    • The "a" tag begins a link, while the "href" property tells the browser where to go once the link is clicked.
    • The text between the "a" tags is what the viewer actually sees and clicks on in order to use the link.
    • The link doesn't have to be associated with text, though. It can be an image, a certain area of the website, or essentially any element!
